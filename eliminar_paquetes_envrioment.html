Deshabilitar el Repositorio Enterprise en Proxmox

Proxmox Virtual Environment (PVE) ofrece una versión empresarial de pago que incluye soporte y acceso a un repositorio especial para actualizaciones. Si estás utilizando la versión gratuita, es recomendable deshabilitar el repositorio empresarial para evitar errores al actualizar o instalar paquetes.

Pasos para deshabilitar el repositorio Enterprise y habilitar el repositorio Community

1. Deshabilitar el Repositorio Enterprise

Por defecto, Proxmox incluye el repositorio empresarial (enterprise) en su configuración de apt. Para deshabilitar este repositorio:

  1. Conéctate al servidor Proxmox a través de SSH o abre una terminal en el servidor.

  2. Edita el archivo del repositorio empresarial. Usa el siguiente comando:

    nano /etc/apt/sources.list.d/pve-enterprise.list

  3. Comenta la línea del repositorio empresarial agregando un # al inicio de la línea, para que quede así:

    #deb https://enterprise.proxmox.com/debian/pve bookworm pve-enterprise

  4. Guarda el archivo y cierra el editor (en nano, usa CTRL+O para guardar y CTRL+X para salir).

2. Añadir el Repositorio Community

A continuación, debes añadir el repositorio comunitario (Community Repository), que es gratuito y se puede usar sin soporte oficial.

  1. Edita el archivo de repositorios de apt:

    nano /etc/apt/sources.list

  2. Añade la siguiente línea al archivo para habilitar el repositorio Community (reemplazando bookworm según tu versión de Debian si es diferente):

    deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription

  3. Guarda y cierra el archivo (CTRL+O y CTRL+X en nano).

3. Verificar el archivo de configuración de repositorios de Ceph

Revisa si tienes configurado el repositorio de Ceph para la versión de pago de Proxmox en los archivos de configuración de apt.

  1. Edita el archivo que contiene el repositorio de Ceph:

    nano /etc/apt/sources.list.d/ceph.list

  2. Si ves una línea como esta (o similar) en el archivo:

    deb https://enterprise.proxmox.com/debian/ceph-quincy bookworm main

    Debes comentarla agregando un # al inicio de la línea, de modo que quede así:

    #deb https://enterprise.proxmox.com/debian/ceph-quincy bookworm main

  3. Guarda el archivo y cierra el editor (CTRL+O para guardar, CTRL+X para salir).

4. Actualizar la lista de paquetes

Después de haber realizado estos cambios, ejecuta el siguiente comando para actualizar los repositorios y verificar que ya no se produzca el error:

apt update

5. (Opcional) Añadir el Repositorio Ceph Community

Si necesitas Ceph y quieres usar el repositorio de la comunidad en lugar del de Enterprise, puedes añadir el repositorio de Ceph Community.

  1. Edita tu archivo de fuentes:

    nano /etc/apt/sources.list

  2. Añade la siguiente línea para el repositorio de Ceph Community (para Quincy, ajusta según la versión de Ceph que necesites):

    deb http://download.proxmox.com/debian/ceph-quincy bookworm main

  3. Guarda y cierra el archivo (CTRL+O, CTRL+X).

  4. Actualiza los repositorios:

    apt update

Conclusión

Siguiendo estos pasos, Proxmox solo utilizará los repositorios gratuitos y no intentará instalar paquetes del repositorio empresarial. Además, opcionalmente, puedes ocultar el mensaje de suscripción no válida en la interfaz web.
